### 删除节点的具体行为

**删除操作流程：**
```
用户点击“删除”
↓
弹出确认对话框
↓
┌───────────────────────────────────────┐
│  ⚠️ 确定要删除此节点吗？                │
│                                       │
│  这将删除：                            │
│  - 节点 "沟通方法" 及所有子节点         │
│  - 所有关联的对话记录                   │
│                                       │
│  此操作不可撤销。                       │
│                                       │
│  [ 取消 ]  [ 确认删除 ]               │
└───────────────────────────────────────┘
↓
确认后执行：
  - 删除当前节点
  - 递归删除所有子节点
  - 删除所有关联的对话历史（通过 messages 表的 chat_id 关联）
  - 更新数据库和缓存
```

```
│                                       │

确认后执行：
  - 删除整个树形结构
  - 删除所有关联的对话数据（通过 messages 表的 chat_id 关联）
  - 应用回到空白状态
  - 显示“新建话题”入口
```

### 数据结构

```typescript
// 节点数据结构
interface TreeNode {
  id: string;                    // 唯一 ID（与原有 chats 表 id 一致）
  title: string;                 // 节点标题（原 chats 表 topic）
  parentId: string | null;       // 父节点 ID（null 表示根节点）
                                 // 通过 parentId 维护树形结构
  order: number;                 // 排序序号
  roleId: string;                // 关联的模型角色 ID（不可为空，有默认角色）
  createdAt: string;             // 创建时间
  updatedAt: string;             // 更新时间
}

// 对话与节点的关联
// 通过 messages 表的 chat_id 字段关联
// chat_id 对应 TreeNode 的 id（与原有 chats 表 id 一致）
interface Message {
  id: string;
  chat_id: string;               // 关联的 TreeNode id（原 chats 表 id）
  role: 'user' | 'assistant';
  content: string;
  created_at: string;
}

// 数据库表结构
interface TreeNodeTable {
  id: TEXT PRIMARY KEY;          // 与原有 chats 表 id 一致
  title: TEXT NOT NULL;          // 对应原 chats 表 topic
  parent_id: TEXT DEFAULT NULL;  // 可为空（根节点为空）
  order: INTEGER DEFAULT 0;
  role_id: TEXT NOT NULL DEFAULT 'default_assistant';  // 不可为空，有默认角色
  created_at: TEXT NOT NULL;
  updated_at: TEXT NOT NULL;
}

// 默认角色配置
const DEFAULT_ROLE_ID = 'default_assistant';
const DEFAULT_ROLE = {
  id: 'default_assistant',
  name: '默认助手',
  description: '一位乐于助人的AI助手',

};
```

### 数据迁移方案

**从 chats 表迁移到 TreeNodeTable：**
```sql
-- 1. 创建 TreeNodeTable 表
CREATE TABLE IF NOT EXISTS tree_nodes (
  id TEXT PRIMARY KEY,
  title TEXT NOT NULL,
  parent_id TEXT DEFAULT NULL,
  order INTEGER DEFAULT 0,
  role_id TEXT NOT NULL DEFAULT 'default_assistant',
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL,
  FOREIGN KEY (parent_id) REFERENCES tree_nodes(id)
);

-- 2. 迁移 chats 表数据到 tree_nodes
--    所有原有对话都成为根节点（parent_id = NULL）
--    保持 id 不变，确保 messages 表的 chat_id 关联正常
INSERT INTO tree_nodes (id, title, parent_id, order, role_id, created_at, updated_at)
SELECT 
  id,
  topic AS title,
  NULL AS parent_id,           -- 所有原有数据都是根节点
  0 AS order,
  'default_assistant' AS role_id,
  created_at,
  updated_at
FROM chats;

-- 3. 验证迁移数据
SELECT COUNT(*) FROM tree_nodes;           -- 应等于原有 chats 记录数
SELECT COUNT(*) FROM messages m            -- 所有 messages 的 chat_id 应在 tree_nodes 中存在
LEFT JOIN tree_nodes tn ON m.chat_id = tn.id
WHERE tn.id IS NULL;                       -- 应为 0
```

**迁移后数据关系：**
```
迁移前：
chats 表                      messages 表
┌──────────────┐              ┌─────────────────────┐
│ id: "chat_1" │ ←───────────→│ chat_id: "chat_1"    │
│ topic: "..."  │              │ ...                  │
│ ...           │              └─────────────────────┘
└──────────────┘

迁移后：
tree_nodes 表                  messages 表
┌─────────────────────┐       ┌─────────────────────┐
│ id: "chat_1"         │ ←───→│ chat_id: "chat_1"    │
│ title: "..."         │       │ ...                  │
│ parent_id: NULL      │       └─────────────────────┘
│ role_id: "default"   │
│ ...                  │
└─────────────────────┘
```

### 对话列表数据来源变更

**原有逻辑：** `SELECT * FROM chats ORDER BY updated_at DESC`

**新逻辑：** `SELECT * FROM tree_nodes WHERE parent_id IS NULL ORDER BY updated_at DESC`

```typescript
// 查询对话列表（侧边栏）
const getRootNodes = async (): Promise<TreeNode[]> => {
  const result = await db.execute(
    `SELECT * FROM tree_nodes 
     WHERE parent_id IS NULL 
     ORDER BY updated_at DESC`
  );
  return result.rows;
};

// 查询某个节点的对话历史
const getMessagesByNodeId = async (nodeId: string): Promise<Message[]> => {
  const result = await db.execute(
    `SELECT * FROM messages 
     WHERE chat_id = ? 
     ORDER BY created_at ASC`,
    [nodeId]
  );
  return result.rows;
};
```

### 视图切换实现细节

```
对话视图 → 思维图视图 切换逻辑：
1. 用户点击“思维图视图”标签
2. 对话区域隐藏，树状思维图显示
3. 当前选中的节点在树中高亮显示（如果有）
4. 点击树节点 → 进入该节点的对话视图

思维图视图 → 对话视图 切换逻辑：
1. 用户点击“对话视图”标签
2. 树状思维图隐藏，对话区域显示
3. 对话区域显示当前选中节点的对话
4. 如果没有选中节点，显示根节点的对话
```
